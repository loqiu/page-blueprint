# Output Template Selector

Choose the concrete blueprint template before writing anything. Do not draft against a generic shared skeleton.

## Selection Rule

- Use `output-template-safe-refactor.md` when the user wants structural cleanup, alignment to existing patterns, or a controlled refactor on top of the current page logic.
- Use `output-template-visual-redesign.md` when the user wants visible redefinition, a new reading path, different module hierarchy, or a shell decision of `bypass`.

## Hard Rule

- `改造级别` is not a label added after the fact.
- The selected template must change the blueprint structure itself.
- If the user has not made the intent clear enough to choose one template, stop and clarify before drafting.
